# Security Policy

## Supported Versions

This project is under active development. Security fixes are applied to the latest version on `main`.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead, report privately with:

- A clear description of the issue
- Reproduction steps
- Impact assessment
- Optional proof-of-concept

If you do not have a private channel configured yet, create a minimal issue without sensitive details and request a private contact path.

## Response Expectations

- Initial triage target: within 7 days
- Status updates provided during investigation
- Fix and disclosure timing based on severity and user impact
